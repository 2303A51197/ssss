Test Driven Development (TDD) – Overdue Fine Calculation

Short Answer (Steps)

  1. Write a failing test (RED): Create the first test case before writing code.
  2. Write the minimum code to pass (GREEN): Implement only simple logic.
  3. Refactor: Clean and optimize code without breaking tests.
  4. Add more test cases: Repeat Red → Green → Refactor.

Test Cases

  • TC1: days=3, rate=10 → Output=30
  • TC2: days=0, rate=10 → Output=0
  • TC3: days=-2, rate=10 → Output=0
  • TC4: days=5, rate=0 → Output=0
  • TC5: days=10, rate=2 → Output=20

Step 1: Write Failing Test (RED)

// RED – Test must fail first
describe("calculateFine", () => {
  it("returns 30 for 3 days overdue at 10/day", () => {
    expect(calculateFine(3, 10)).toBe(30);
  });
});

Step 2: Minimum Code to Pass (GREEN)

function calculateFine(days, rate) {
  return days * rate;
}

Step 3: Add More Test Cases

it("returns 0 when days = 0", () => {
  expect(calculateFine(0, 10)).toBe(0);
});

it("returns 0 for negative days", () => {
  expect(calculateFine(-2, 10)).toBe(0);
});

it("returns 0 when rate = 0", () => {
  expect(calculateFine(5, 0)).toBe(0);
});

Step 4: Refactored Code (Final)

function calculateFine(days, rate) {
  if (days < 0 || rate < 0) return 0;
  return days * rate;
}
